# Review: S2-04 Google Sheets Repository

## Summary
Implementation of `GoogleSheetsTransactionRepository`, the first concrete repository fulfilling the `TransactionRepository` contract.

## Created Files
- `src/config/Config.gs`
- `src/repository/GoogleSheetsTransactionRepository.gs`
- `docs/engineering/reviews/S2-04.md`

## Design Decisions
- **Extends Abstract Contract:** `GoogleSheetsTransactionRepository` implements the `save(transaction)` method required by `TransactionRepository`.
- **Infrastructure Isolation:** Access to `SpreadsheetApp` is restricted solely to this repository class, keeping domain logic clean.
- **Configurable:** Spreadsheet and sheet identifiers are managed via a central `Config` object to avoid hardcoding and improve maintainability.

## Repository Flow
1. Receives a `Transaction` domain object.
2. Opens the configured spreadsheet using `SpreadsheetApp`.
3. Retrieves the designated Ledger sheet.
4. Appends a new row containing the transaction data in the specified field order.

## Error Handling
- Throws an error if the transaction object is missing.
- Throws an error if the spreadsheet cannot be opened.
- Throws an error if the ledger sheet cannot be found.
- Throws an error if the `appendRow` operation fails.

## Ready for Review
The repository is implemented as a simple, stateless class for appending transactions, fulfilling the S2-04 requirements.

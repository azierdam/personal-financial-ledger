# Implementation Plan: C2.1 — Google Sheets Repository

## Objective
Implement storage-agnostic persistence adapters for `Account`, `Category`, and `Transaction` entities using Google Sheets while ensuring strict architectural isolation.

## Key Changes
- `src/infrastructure/sheets/mappers/`: New package for domain-to-row mapping logic.
- `src/infrastructure/sheets/repositories/`: New package for concrete repository implementations.
- `test/integration/SheetsPersistenceTests.gs`: Integration tests using a test spreadsheet (or mock).

## Implementation Steps
1. Create directory structure `src/infrastructure/sheets/{mappers,repositories}`.
2. Implement Mappers (e.g., `TransactionMapper`) for all three entities.
3. Implement `SheetsAccountRepository`, `SheetsCategoryRepository`, `SheetsTransactionRepository`.
4. Ensure `TransactionRepository` only implements `save`, `findById`, `findAll` (no delete/update).
5. Implement integration tests to verify persistence logic works as expected.

## Verification
- Run integration tests to confirm CRUD operations function correctly for all repositories.
- Code review to ensure zero leakage of `SpreadsheetApp` into `src/domain/` or `src/repository/`.
- Ensure immutability for `Transaction` is enforced in the infrastructure layer.

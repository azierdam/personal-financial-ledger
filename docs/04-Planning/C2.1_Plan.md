# Implementation Plan: C2.1 — Google Sheets Repository

## Objective
Implement storage-agnostic persistence adapters for `Account`, `Category`, and `Transaction` using `SheetsGateway` and explicit Mappers.

## Key Changes
- `src/infrastructure/sheets/SheetsGateway.gs`: Lightweight gateway for SpreadsheetApp.
- `src/infrastructure/sheets/Errors.gs`: Custom infrastructure exceptions.
- `src/infrastructure/sheets/mappers/`: `AccountMapper`, `CategoryMapper`, `TransactionMapper`.
- `src/infrastructure/sheets/repositories/`: `SheetsAccountRepository`, `SheetsCategoryRepository`, `SheetsTransactionRepository`.
- `test/integration/SheetsRepositoryTest.gs`: Integration tests (using fakes/mocks for spreadsheet).
- `docs/03-Engineering/Infrastructure_Architecture.md`: Document infrastructure and mapping design.

## Implementation Steps
1. Create directory structure `src/infrastructure/sheets/{mappers,repositories}`.
2. Implement `Errors.gs` for infrastructure-specific exceptions.
3. Implement `SheetsGateway.gs`.
4. Implement Mappers (e.g., `TransactionMapper`) for all three entities.
5. Implement `SheetsAccountRepository`, `SheetsCategoryRepository`, `SheetsTransactionRepository`.
6. Implement integration tests to verify persistence logic works as expected.

## Verification
- Run integration tests to confirm CRUD operations function correctly for all repositories.
- Code review to ensure zero leakage of `SpreadsheetApp` into `src/domain/` or `src/repository/`.
- Ensure immutability for `Transaction` is enforced in the infrastructure layer.

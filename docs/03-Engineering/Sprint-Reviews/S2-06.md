# Review: S2-06 Telegram Adapter + Currency Configuration Fix

## Summary
Implementation of the `TelegramAdapter` for webhook orchestration and refinement of `TransactionService` to use configurable currency.

## Created Files
- `src/adapter/TelegramAdapter.gs`
- `docs/03-Engineering/Sprint-Reviews/S2-06.md`

## Modified Files
- `src/config/Config.gs`
- `src/service/TransactionService.gs`

## Architecture Flow
Telegram Webhook -> TelegramAdapter -> TransactionParser -> TransactionService -> TransactionRepository -> Google Sheets

## Currency Fix
Replaced hardcoded 'USD' in `TransactionService` with `Config.getDefaultCurrency()`, which retrieves the value from `ScriptProperties` with a fallback to 'IDR'.

## Design Decisions
- **Adapter Pattern:** `TelegramAdapter` decouples Telegram webhook processing from application business logic.
- **Configurable Infrastructure:** Currencies are now configurable via `ScriptProperties` instead of being hardcoded.
- **Orchestration:** `TelegramAdapter` correctly handles the full transaction lifecycle by coordinating the Parser and Service components.

## Error Handling
- `TelegramAdapter` catches all processing errors and returns a 500 JSON response, or a 400 for invalid payloads.

## Ready for Review
The adapter is implemented, and currency configuration is fixed, fulfilling the S2-06 requirements.

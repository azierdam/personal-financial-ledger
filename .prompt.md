# Implementation Task: Refactor the Engineering CLI workflow so that Git repository operations and working file generation are separated into distinct commands.

The objective is to simplify the workflow, eliminate unnecessary Working Tree Safety conflicts, and make each command responsible for a single phase of the engineering lifecycle.

## Sprint / Milestone
Engineering CLI – Workflow Separation

## Scope
Design and implement a two-phase workflow.

## Phase 1 — Repository Preparation

Introduce a dedicated command responsible only for repository operations.

Responsibilities:

- validate repository
- checkout main
- pull latest changes
- create feature branch
- switch to feature branch

This command must not generate any working files.

## Phase 2 — Working File Generation

Refactor the existing `prepare` command.

Responsibilities:

- validate required engineering artifacts
- generate `.context.md`
- generate `.prompt.md`

This command must not:

- checkout branches
- pull from Git
- create branches
- modify repository state

It operates only on the current branch.

## Constraints
- Preserve backward compatibility where practical.
- Keep the Engineering CLI simple.
- One command should have one primary responsibility.
- Do not modify the approval parser.
- Do not modify package generation.
- Do not introduce unnecessary dependencies.

## Acceptance Criteria
The sprint is complete when:

- Repository preparation is separated from working-file generation.
- `prepare` no longer performs Git checkout or branch creation.
- Repository operations occur in a dedicated command.
- Documentation reflects the new workflow.
- Automated tests are updated.
- Existing engineering workflow remains functional.

## Deliverables
- Workflow refactoring
- Updated documentation
- Implementation Summary
- Validation Results
- Git Status
- Recommended Commit Message

## Conventional Commit
refactor(cli): separate repository preparation from working file generation

## Stop Condition
Stop after implementation.

Generate the Engineering Review Package using:

python -m tools.engineering package chatgpt

Do not perform repository finalization.

Wait for Technical Lead review.
